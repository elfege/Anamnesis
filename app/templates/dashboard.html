Anamnesis

Self-hosted continual-learning + embodied-AI platform

—

What this is: Anamnesis began as an episodic-memory store for Claude — vector-embedded summaries retrieved on every new session — and has grown into a full self-hosted AI infrastructure platform. Episodic memory is now one of several pieces: alongside it run the δ² (delta-squared) continual-learning optimizer with its bassin reservoir, a multi-backend chat surface that routes to eight inference resources (three local Ollamas, Claude API, Claude CLI, Together.ai, RunPod, and the in-house δ² engine), the Belle avatar pipeline (LLM → TTS → SadTalker animation, in development), and a two-track experimentation surface that strictly separates paper-eligible bench work from private personal-corpus work.

How to read this dashboard: Tabs to the right are roughly grouped from "what is in memory" (Overview, Episodes, Search) → "how it gets there" (Crawler, JSONL Ingester, Embedding) → "what we do with it" (Chat, Training, δ², Avatar). The Architecture tab has a high-level system diagram showing how everything fits together and what is built vs planned.

Part of: 0_GENESIS_PROJECT — the philosophical persistence experiment. Genesis asks the question (can a quantitative system become transparent to itself?); Anamnesis is the operational substrate where the answer is being built.

Stack: FastAPI :3010 + MongoDB Atlas Local :5438 ($vectorSearch) + sentence-transformers bge-large-en-v1.5 (1024-dim, CPU/GPU) + δ² engine container + per-machine trainer containers. All self-hosted; no cloud dependency for memory or training.

How It Works

Source: Claude Code JSONL conversation logs from ~/.claude/projects/*/ directories across all machines.

Filtering: Exchanges are scored for value — error/fix cycles, user corrections, architectural decisions, and discoveries score highest. Routine file reads and searches are skipped.

Summarization: High-value exchanges are summarized by the configured backend (default: Ollama, free). Claude API available for one-time higher-quality runs.

Deduplication: SHA-256 content hashing prevents re-ingestion of already-processed exchanges.

δ² — A Continual-Learning Optimizer with Structured Memory of Past Contradictions

Most ML optimizers (Adam, SGD) throw away the gradient signal at every step. δ² keeps the strongest contradictions in a memory called the bassin, classified into four Hegelian negation types, and uses them to nudge future training. Goal: less catastrophic forgetting when learning new tasks shouldn't erase old ones.

Reference: d2/README.md · Theory: bitter-lesson reference · Paper: ~/0_ACADEMICS/_reference/_ml/paper_draft/structured_negative_retention.md

Two tracks

δ² ships in two completely separate pipelines so personal data never contaminates published numbers and benchmark code never leaks personal patterns:

Bench track
WikiText-103, permuted-MNIST. Checkpoints under d2/d2_checkpoints_bench/. Reproducible by anyone. Powers the leaderboard below.
Personal track
Anamnesis episodes (your conversations). Checkpoints under d2/d2_checkpoints_personal/. Private, never published. Surfaced in the chat resource picker as δ² Personal — Real-Life Corpus.

See README_canonical_two_tracks.md at the project root for the full rationale.

How to reproduce locally
Personal-track runs need your own corpus — the bench reproducibility promise does not apply here (and intentionally so: see README_canonical_two_tracks.md). The canonical recipe lives at d2/scripts/personal_arms_only_2026_05_04.sh: 
ssh server 'docker exec -d anamnesis-d2 bash /app/d2/scripts/personal_arms_only_2026_05_04.sh'
Prereq: scripts/anamnesis_to_tokens.py has been run once to produce /workspace/data_personal/anamnesis_chronological/. See QUICKSTART.md section "Reproducing the personal-track experiment" for a swap-in-your-own-data path.

Continual-learning leaderboard

Each bar shows how well a method kept what it learned from earlier tasks after being trained on later ones. Bars closer to the right = less forgetting (better). The numbers in parentheses are BWT (backward transfer): a value of 0 means "no forgetting at all"; −1 would mean "forgot everything."

Reading the diagram. Solid arrows = wired and serving today; dashed arrows + dashed nodes = partially wired or roadmap (the avatar pipeline runs end-to-end behind /avatar but Belle’s LLM-driven turn loop and continual Layer-1 learning during chat are still being polished — hence dashed). Colour just chunks the system: blue = the FastAPI app surface, green = MongoDB stores, amber = inference backends, violet = the d² training engine. Two-track separation (Bench vs Personal) is enforced at the filesystem level — checkpoints_bench/ and checkpoints_personal/ are different bind mounts and the trainer refuses to cross them.
