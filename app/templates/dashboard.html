Anamnesis

Episodic Memory for Claude Instances

—

What this is: A vector embedding memory system for Claude. Each session, significant experiences are distilled into text, converted to 1024-dimensional vectors by a local sentence-transformers model (bge-large-en-v1.5), and stored in MongoDB. Next session, the current context is embedded, the most relevant past episodes are retrieved by cosine similarity, and loaded into context. The AI remembers — imperfectly, selectively, but non-randomly.

Part of: 0_GENESIS_PROJECT — the philosophical persistence experiment. Genesis asks the question; Anamnesis is the first technical answer.

Stack: FastAPI + MongoDB Atlas Local (vector search) + sentence-transformers bge-large-en-v1.5 (1024 dims, CPU/GPU) | Port 3010 (API) + 5438 (Mongo)

How It Works

Source: Claude Code JSONL conversation logs from ~/.claude/projects/*/ directories across all machines.

Filtering: Exchanges are scored for value — error/fix cycles, user corrections, architectural decisions, and discoveries score highest. Routine file reads and searches are skipped.

Summarization: High-value exchanges are summarized by the configured backend (default: Ollama, free). Claude API available for one-time higher-quality runs.

Deduplication: SHA-256 content hashing prevents re-ingestion of already-processed exchanges.
